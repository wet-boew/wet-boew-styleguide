Travailler derrière un proxy

Les étapes suivantes peuvent être utilisées pour ceux qui travaillent derrière un proxy qui requiert une authentification. Toutes ces méthodes vont emmagasiner votre mot de passe en texte clair, alors faites attention si vous utilisez l'une des méthode.

Configurer le proxy pour une installation NPM

  1. Ouvrir une invite de commande
  2. Exécuter les commandes suivantes en remplacant USAGER, MOT_DE_PASSE, ADRESSE_PROXY, et PORT_PROXY avec les informations de votre réseau :
    1. npm config set proxy http://USAGER:MOT_DE_PASSE@ADRESSE_PROXY:PORT_PROXY
    2. npm config set https-proxy http://USAGER:MOT_DE_PASSE@ADRESSE_PROXYS:PORT_PROXY

Configurer le proxy pour Git

  1. Ouvrir une invite de commande
  2. Exécuter les commandes suivantes en remplacant USAGER, MOT_DE_PASSE, ADRESSE_PROXY, et PORT_PROXY avec les informations de votre réseau :
    1. git config --global --add proxy http://USAGER:MOT_DE_PASSE@ADRESSE_PROXY:PORT_PROXY
    2. git config --global --add https-proxy http://USAGER:MOT_DE_PASSE@ADRESSE_PROXYS:PORT_PROXY
  3. Exécuter les commandes suivantes pour prévenir des problèmes reliés à SSH :
    1. git config --global url."https://".insteadOf git://
    2. git config --global url."https://github.com/".insteadOf git@github.com: (don't remove the final colon)
  4. Si votre ordinateur fait parti d'un réseau corporatif qui utilise des espaces réseau personnels, assurez-vous que des copies identiques du fichier .gitconfig existent sur votre espace personnel et sur votre profil personnel local (exemple : F:\ et C:\Users\VotreUtilisateur)

Configurer le proxy pour d'autres tâches

Plusieurs modules d'extension vérifient aussi pour les variables environnementales PROXY_HTTP et PROXY_HTTPS lorsq'ils ont besoin de faire un appel extérieur à l'Internet.

  1. Vous pouvez configurer ces variables avec la valeur http://USAGER:MOT_DE_PASSE@ADRESSE_PROXY:PORT_PROXY à chaque session en apellant :
    1. Invite de commande Windows : set HTTP_PROXY=http://USAGER:MOT_DE_PASSE@ADRESSE_PROXY:PORT_PROXY
    2. Bash Shell : export HTTPS_PROXY=http://USAGER:MOT_DE_PASSE@ADRESSE_PROXY:PORT_PROXY
  2. Pour éviter de faire cet action à chaque fois, vous pouvez ajouter ces commandes à votre fichier console "RC". Si vous utilisez le Git Bash Shell sur Windows, vous pouvez ajouter les déclarations d'exportation au fichier ".bashrc" à la base de votre profil d'usager (C:\Users\USAGER). Ce fichier est exécuté chaque fois que vous ouvrez une session Git Bash, alors les variables environnementales seront disponibles de l'invite de commande. Vous devrez peut être créer ce fichier a vec un éditeur de texte, puisque Windows ne permet pas de créer des fichiers débutant par un point à travers Windows Explorer.
