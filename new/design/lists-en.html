Description

There are three list types:

  • Unordered lists - A list of items in which the order does not explicitly matter.
  • Ordered lists - A list of items in which the order does explicitly matter.
  • Description lists - A list of terms with their associated descriptions.

Default appearance

The default appearance of the three list types are as follows:

Unordered lists

A list of items in which the order does not explicitly matter.

  • Lorem ipsum dolor sit amet
  • Nulla volutpat aliquam velit
    • Phasellus iaculis neque
    • Purus sodales ultricies
    • Vestibulum laoreet porttitor sem
  • Faucibus porta lacus fringilla vel
  • Aenean sit amet erat nunc
  • Eget porttitor lorem
<ul>
  <li>...</li>
</ul>

Ordered lists

A list of items in which the order does explicitly matter.

  1. Lorem ipsum dolor sit amet

  3. Faucibus porta lacus fringilla vel
  4. Aenean sit amet erat nunc
  5. Eget porttitor lorem

  <li>...</li>
</ol>

Horizontal description lists

Make terms and descriptions in <dl> line up side-by-side. Starts off stacked like default <dl>s, but when the navbar expands, so do these.

Item 1
Description of item 1
Item 2
Description of item 2
Item 3
Description of item 3
<dl class="dl-horizontal">
  <dt>...</dt>
  <dd>...</dd>
</dl>

Auto-truncating

Horizontal description lists will truncate terms that are too long to fit in the left column with text-overflow. In narrower viewports, they will change to the default stacked layout.

List columns

A lists can be spread over 2, 3 or 4 columns using the CSS attribute column-count. Since columns are similar to grids, they adhere to the same breakpoints as grids. Therefore, the classes available are:

  • clmn-2-xs, clmn-2-sm, clmn-2-md, clmn-2-lg
  • clmn-3-xs, clmn-3-sm, clmn-3-md, clmn-3-lg
  • clmn-4-xs, clmn-4-sm, clmn-4-md, clmn-4-lg

Cross-browser compatibility

Internet Explorer 9 and below does not support the column-count attribute. In those browsers the list will linearize as one list in one column.
