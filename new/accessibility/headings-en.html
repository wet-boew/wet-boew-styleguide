Description

A heading is a short text phrase which identifies the section of content that immediately follows it. Heading elements are <h1>-<h6>.

The page title is a short text phrase which identifies the contents of the web page. The title element is <title>.

Headings

Related to success criteria

1.3.1 Info and Relationships
2.4.6 Headings and Labels

When using headings:

  • Identify heading text using heading elements (<h1>-<h6>).

    • Start at heading element <h1> (recommended).
    • Don't skip heading element levels (e.g <h2>, <h4>) (recommended).

  • Use descriptive text to identify and differentiate the contents of its section from other content on the web page.

Important: Enlarge non-heading text using only CSS. Do not use heading elements.

Page titles

Related to success criteria

2.4.2 Page Titled

When using page titles:

  • Use the <title> element.

  • Use descriptive text to identify and differentiate the contents of the web page from other content of other web pages.

Note: It is sufficent for a web application to use only its web application title on all of its web screens.

Accessibility considerations

Cognitive impairments

Separate information so it is logically ordered, organized, and consistent. A properly formatted document gives users quick access, and a thorough understanding. This enhances everyone's user experience, not just those with cognitive impairments.

Visual impairments

Screen readers allow users to jump from heading to heading, as well as navigate between different levels of heading elements. Web pages that are properly structured with <h1> to <h6> headings are much easier to navigate by screen readers.
